# 1) Problem understanding

## Key issues in the current system (typical pain points)

- **First-time anxiety:** users don’t know *what happens next*, what is required, or how long it will take.
- **Long forms:** too many fields at once → fatigue and abandonment.
- **Unclear requirements:** document rules are often vague (“upload ID proof”) without examples.
- **Navigation breaks:** users lose context when they move between form → uploads → booking.
- **No save confidence:** drafts may exist but users don’t trust the system saved their work.

## What frustrates users

- Re-entering the same data after errors or timeouts
- Not knowing whether something is “required” or “optional”
- Booking appointments after investing time, then finding no slots
- Generic error messages that don’t say what to do

## Where confusion / drop-offs happen

- **During form filling:** when the page is long and the user can’t estimate remaining effort
- **Documents:** when requirements don’t match what the user has
- **Appointments:** when users see lots of “full” slots or unclear center options
- **Submission:** when users can’t confirm what will be generated/available afterward (ID, receipt, PDF)
